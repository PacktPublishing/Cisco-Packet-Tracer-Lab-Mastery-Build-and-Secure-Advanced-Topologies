How to Set Up SSH and Console Passwords
Securing access to your Cisco switches and routers is one of the first and most important tasks when deploying network devices. This article guides you through the step-by-step process of setting up both SSH (Secure Shell) and Console passwords to help protect your infrastructure from unauthorized access.



ðŸ” Why Secure Console and SSH Access?
Console access provides direct management via a physical connection.

SSH access allows for encrypted remote administration over the network.

Without proper passwords and encryption in place, your network could be vulnerable to intrusion, misconfiguration, or even a complete takeover.

Step 1: Configure the Domain Name
SSH requires a domain name for key generation.

Switch> enable
Switch# configure terminal
Switch(config)# ip domain-name example.com
Step 2: Create a Local User Account
This account will be used to authenticate SSH sessions.

Switch(config)# username admin privilege 15 secret MySecurePassword
secret encrypts the password. Avoid using password in production environments.

Step 3: Enable SSH and Generate Crypto Keys
Switch(config)# crypto key generate rsa
When prompted, choose a key size of at least 2048 bits for better security.

Now, enable the SSH version:

Switch(config)# ip ssh version 2
Step 4: Configure the VTY (Virtual Terminal) Lines for SSH Access
Switch(config)# line vty 0 4
Switch(config-line)# transport input ssh
Switch(config-line)# login local
Switch(config-line)# exit
This limits remote access to SSH only and uses local user credentials.

Switch(config-line)# transport input ssh

Step 6: Set the Console Line Password
This secures physical access to the device via a console cable.

Switch(config)# line console 0
Switch(config-line)# password ConsolePass123
Switch(config-line)# login
Switch(config-line)# exit
Without the login command, the password wonâ€™t be prompted.

Step 7: Save Your Configuration
Always save your configuration to ensure it persists after a reboot:

Switch# write memory
